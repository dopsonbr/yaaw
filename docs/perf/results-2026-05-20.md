# YAAW perf fixes — results, 2026-05-20

Branch: `perf/baseline-and-fixes`. Five commits land in order on top of the
baseline measurement infrastructure:

1. `Fix #1` — Incremental SQLite persist + thread indexes
2. `Fix #2` — AppModel thread lookup caches
3. `Fix #3` — Public tree builder + debounced file-search input
4. `Fix #4` — Bound lifecycle events and async-log diagnostics
5. `Fix #5` — Fast-forward stale capture-log offsets

Same machine, same fixtures, same release-mode build.

## Headline numbers

| Benchmark | Baseline | After | Change | Targeted by |
| --- | ---: | ---: | --- | --- |
| `save_singleThreadEdit_in10kCorpus` | 100 ms | **2 ms** | **≈60× faster** ✓ | Fix #1 |
| `activeThreadsForSelectedProject_10k` (100 reads) | 225 ms | **0.077 ms** steady | **≈2900× faster** ✓ | Fix #2 |
| `activeThreadsForSelectedProject_1k` (100 reads) | 23 ms | 0.065 ms steady | ≈350× faster ✓ | Fix #2 |
| `selectThread_in_10kCorpus` | 10 ms | 0.33 ms | ≈30× faster ✓ | Fix #1 + #2 |
| `treeBuilder_50k` | 188 ms | **54 ms** | **3.5× faster** ✓ | Fix #3 |
| `treeBuilder_5k` | 10 ms | 7 ms | 1.4× faster | Fix #3 |

These three are the user-perceptible wins: every UI mutation now persists
in 2 ms instead of 100 ms; every SwiftUI thread-list redraw is essentially
free; the 50k-entry tree assembly is 3.5× faster.

**Note on the bulk-save regression**: `save(_:)` still rewrites everything,
and now also maintains two indexes per row. That's expected. The hot
path is no longer `save(_:)` — AppModel routes every mutation through the
narrow UPSERT methods, which is what the `singleThreadEdit` row measures.
Initial-seed cost is one-time and acceptable.

## Stability fixes (no benchmark, but worth noting)

- `PlaceholderTerminalSessionManager.lifecycleEvents` now caps at 500
  entries with FIFO eviction (Fix #4). Long-lived sessions stop leaking
  memory.
- `LoggerDiagnosticEventRecorder` caches `Logger` instances per category
  and emits on a serial utility-qos queue, so `record(_:)` returns
  immediately (Fix #4).
- `capturedOutput` fast-forwards past stale buffered output when the
  reader has fallen >8 MB behind (Fix #5). A flooded agent process no
  longer leaves the UI showing minutes-old text.

## What's NOT addressed

- **Disk-side capture log truncation**: the `script`-managed log can still
  grow within a single agent session. Cleanup happens at next terminal
  launch (file is deleted before re-creation). A real cap requires
  replacing `script` or wrapping its pipe — deferred.
- **AppModel-layer tree memoization**: Fix #3 makes the builder faster
  but each redraw still rebuilds the tree. True per-frame memoization
  needs RootView's expansion state to move into the model. Deferred.
- **File-system watching**: the indexer still walks the whole tree on
  every re-trigger. An FSEvents-backed observer would avoid re-walks.
  Deferred.

## How to reproduce

```sh
git checkout perf/baseline-and-fixes
RUN_BENCHMARKS=1 swift test -c release --filter YAAWKitBenchmarks
```

Each measure block runs 10 iterations; numbers reported above are the
average. RSD ranges from 1% on the heavier benchmarks to ~60% on the
sub-1 ms ones (where measurement granularity dominates).
