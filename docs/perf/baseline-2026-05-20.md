## Read these numbers

- **`save_singleThreadEdit_in10kCorpus = 100 ms`** is the single most important
  number. Every thread rename, panel toggle, terminal expand, navigation step in
  a 10k-thread world pays this. Fix #1 must crush it.
- **`fuzzy_50k_threeChar = 916 ms`** means each character of file-filter input
  blocks the UI for nearly a second on a large repo. Fix #3 (debounce + memoize)
  targets this directly.
- **`activeThreadsForSelectedProject_10k = 2.25 ms/read`** sounds small until you
  remember SwiftUI may read computed properties dozens of times per redraw. Fix
  #2's index cache makes this O(1).
- **`index_largeRepo = 1.5 s`** is acceptable for the cold path (it's on a
  background queue), but Fix #3's memoization avoids paying it on every
  state-change re-trigger.
