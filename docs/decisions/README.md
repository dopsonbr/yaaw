# Decisions

Lightweight architecture and product decision records. Each record uses this template:

- **Status:** Open / Accepted / Superseded.
- **Context:** Background and the constraint that forced the decision.
- **Options:** Choices under consideration when the status is Open.
- **Decision:** What was chosen when the status is Accepted.
- **Consequences:** What the decision commits the project to.

Open decisions are not blockers for unrelated work, but they should be resolved before the dependent area ships. Each open record names a recommended default so plans can proceed without stalling.
